# 基于STM32+ESP8266的IoT环境监测系统

> 简历描述：基于STM32F103C8T6 + ESP8266的IoT环境监测系统，实现I2C传感器驱动、UART AT指令控制WiFi模块、MQTT协议云端数据上报、FreeRTOS多任务调度及低功耗设计。

### 供电说明

```
USB → 最小系统板 → 5V引脚 → AMS1117输入
                            AMS1117输出 → ESP8266 VCC + CH_PD
                  GND引脚 → ESP8266 GND（必须共地）
```

> CH_PD是ESP8266使能脚，必须接3.3V，不能悬空。
> ST-Link只接SWDIO、SWCLK、GND，不接VCC。

## 三、系统流程

```
SHT30
  │ I2C（每5秒）
  ▼
STM32F103C8T6
  ├─ 解析数据 + CRC校验
  ├─ OLED显示温湿度
  ├─ 超阈值？→ 蜂鸣器 + LED报警
  ├─ 存入Flash本地缓存
  │
  │ UART AT指令
  ▼
ESP8266
  │ MQTT
  ▼
云端平台（历史曲线 + 实时数值）
```

断网时数据缓存到Flash，联网后自动补传。

## 四、通信协议

### STM32 ↔ SHT30（I2C）
- 地址：0x44（ADDR接GND）
- 采集命令：0x2C06（单次高精度）
- 返回6字节：温度2字节 + CRC1字节 + 湿度2字节 + CRC1字节

### STM32 ↔ ESP8266（UART）
- 波特率：115200
- 通过AT指令控制：连接WiFi、建立MQTT连接、发布数据

### MQTT数据格式
```json
{
  "temperature": 25.6,
  "humidity": 60.2,
  "timestamp": 1700000000
}
```

## 六、开发流程

### 阶段一：传感器驱动
- SHT30 I2C通信，串口打印温湿度数据
- 验证CRC校验正确

### 阶段二：本地功能
- OLED显示温湿度
- 超阈值蜂鸣器+LED报警
- 按键设置阈值

### 阶段三：WiFi上云
- CH340调试ESP8266，AT指令跑通
- MQTT连接云端，数据上报成功
- 云端面板显示数据

### 阶段四：存储与补传
- Flash本地缓存历史数据
- 断网重连后自动补传

### 阶段五：FreeRTOS重构
- 拆分为三个任务：采集任务、上报任务、显示任务
- 用Queue在任务间传递数据
- 用Semaphore做任务同步

### 阶段六：低功耗
- 无数据变化时进入STOP模式
- RTC定时唤醒采集

### 阶段七：收尾
- 整理代码，驱动层与应用层分离
- 写README，画接线图
- 录演示视频，上传GitHub

## 七、扩展方向

### FreeRTOS多任务（强烈推荐）
嵌入式岗位几乎必问，项目里真正用过比背八股有说服力。
三个核心概念：Task任务、Queue队列、Semaphore信号量。

### 低功耗设计
涉及STM32时钟树、STOP模式、RTC唤醒，IoT和消费电子岗位很看重。

### OTA远程升级
消费电子必备技能，涉及Flash分区、固件校验、回滚机制，进阶方向。

### 扩展传感器
加CO2（MH-Z19）或PM2.5（PMS5003），升级为空气质量监测站。
